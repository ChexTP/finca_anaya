GUIA DE COMPROBACION - PRUEBAS COMPLETAS SISTEMA FINCA ANAYA
Fecha: 22/07/2026

Objetivo:
Verificar el funcionamiento completo del sistema antes de usar datos reales, desde la recepcion de cafe hasta ventas, facturacion interna, pagos, bodega, procesos, muestras, documentos y gerencia.

Recomendacion para la prueba:
- Usar datos de prueba, no datos reales definitivos.
- Probar con cada rol por separado.
- Anotar cualquier pantalla confusa, dato que falte o paso que no coincida con el trabajo diario.
- En cada caso revisar el resultado esperado antes de pasar al siguiente.

2. RECEPCION DE CAFE EN BODEGA

Usuario recomendado: bodega o admin.
Ruta: Recepcion.

Prueba A - Crear proveedor:
1. Abrir Recepcion.
2. Crear un proveedor nuevo con nombre, telefono, direccion y zona.
3. Guardar.

Resultado esperado:
- El proveedor queda disponible para registrar cafe recibido.

Prueba B - Registrar cafe completo:
1. Seleccionar proveedor.
2. Seleccionar metodo de proceso: Lavado, Natural o Semilavado.
3. Poner fecha de llegada a bodega.
4. Poner peso bruto.
5. Seleccionar empaque.
6. Poner cantidad de empaques.
7. Marcar bolsa interna si aplica.
8. Poner humedad.
9. Poner factor de rendimiento.
10. Seleccionar categoria: Regional, Varietal o Exotico.
11. Seleccionar o escribir la clasificacion exacta.
12. Agregar zona de procedencia y observaciones si aplica.
13. Guardar.

Resultado esperado:
- El sistema calcula el peso neto.
- Si tiene humedad y factor, el lote queda pendiente de laboratorio.
- La fecha de llegada, categoria, clasificacion exacta, humedad y factor no se pierden.

Prueba C - Registrar cafe sin humedad o sin factor:
1. Repetir registro, pero dejar vacia humedad o factor.
2. Guardar.

Resultado esperado:
- El lote queda en Pendientes de revision fisica.
- Bodega puede registrar despues humedad y factor.
- Al completar revision fisica, el lote pasa a Pendientes de laboratorio.

Prueba D - Validacion de clasificacion:
1. Intentar registrar Regional, Varietal o Exotico sin clasificacion exacta.

Resultado esperado:
- El sistema no debe dejar guardar.
- Debe pedir clasificacion, variedad o codigo exacto.


3. LABORATORIO - REVISION DE LOTES RECIBIDOS

Usuario recomendado: laboratorio o admin.
Ruta: Laboratorio.

Prueba:
1. Abrir Laboratorio.
2. Seleccionar un lote pendiente.
3. Revisar que aparezcan los datos recibidos desde bodega.
4. Poner catacion: aroma, fragancia, sabor, dulzor, cuerpo, residual, taza limpia y score.
5. Poner notas de laboratorio.
6. Aprobar el lote.

Resultado esperado:
- El lote queda disponible en inventario.
- El sistema no pide acidez, balance ni uniformidad.
- Si laboratorio cambia categoria o clasificacion, debe pedir nota interna.
- El historial debe conservar el cambio.

Prueba de rechazo:
1. Seleccionar otro lote pendiente.
2. Marcar como rechazado.

Resultado esperado:
- El lote queda rechazado.
- Bodega puede verlo para retiro si corresponde.

12. PROCESOS

Usuario recomendado: bodega o admin.
Ruta: Procesos.

Prueba:

2. Seleccionar la venta.
3. Seleccionar lotes y cantidades que se llevan a proceso.
4. Crear proceso.

Resultado esperado:
- Se crea proceso con codigo PRO.
- La venta pasa a proceso solicitado.
- El proceso queda pendiente de iniciar.

Prueba de ciclo de proceso:
1. Marcar proceso como iniciado.
2. Poner ubicacion y fecha estimada de regreso.
3. Marcar que regreso para revision fisica.
4. Registrar cantidad final, humedad y factor.
5. En laboratorio completar examen del proceso.

Resultado esperado:
- Mientras esta en proceso, bodega puede ver el estado.
- No vuelve a bodega como disponible hasta tener datos fisicos y de laboratorio.
- Al finalizar, se crea lote PROC disponible.

Prueba de busqueda:
1. Buscar proceso por codigo, cliente, venta, lote, perfil o ubicacion.

Resultado esperado:
- La lista se filtra correctamente.

22. COMPROBACION FINAL DE FLUJO COMPLETO

Flujo recomendado para prueba integral:

1. Bodega registra proveedor.
2. Bodega registra cafe recibido.
3. Si falta humedad/factor, bodega completa revision fisica.
4. Laboratorio aprueba cafe.
5. Inventario muestra lote disponible.
6. Admin configura perfil comercial con receta si es exotico.
7. Vendedor crea cliente.
8. Vendedor crea cotizacion con varios cafes.
9. Contabilidad acepta cotizacion y convierte a venta.
10. Contabilidad registra pago inicial o pendiente.
11. Bodega ve venta en Pedidos.
12. Bodega asigna prioridad y encargado.
13. Si hay stock, bodega asigna lote.
14. Si no hay stock, bodega marca "No hay" o solicita proceso.
15. Bodega crea proceso si aplica.
16. Bodega inicia proceso y registra regreso fisico.
17. Laboratorio revisa proceso.
18. Bodega define ensamble si aplica.
19. Bodega imprime orden.
20. Bodega marca alistada.
21. Inventario descuenta cantidades.
22. Bodega marca despachada.
23. Contabilidad registra abonos si quedan saldos.
24. Documento de venta muestra pagos.
25. Gerencia ve datos de ventas, procesos, deficit y faltantes.

Resultado esperado:
- El flujo completo se puede realizar sin usar Excel para controlar estados principales.
- Cada rol ve la informacion que necesita.
- Los datos clave no se pierden entre modulos.
- El sistema ayuda a decidir que cafe hay, que cafe falta y que pedido sigue.


23. OBSERVACIONES PARA QUIEN PRUEBA

Anotar por cada problema:
- Usuario con el que estaba probando.
- Modulo.
- Codigo del lote, venta, muestra o proceso.
- Que estaba intentando hacer.
- Que esperaba que pasara.
- Que paso realmente.
- Captura de pantalla si es posible.
- Si el problema impide seguir el flujo o solo es una mejora visual.

Prioridad sugerida:
- Alta: bloquea el trabajo o causa calculos incorrectos.
- Media: permite trabajar, pero confunde o genera pasos extra.
- Baja: mejora visual, texto, orden o comodidad.
